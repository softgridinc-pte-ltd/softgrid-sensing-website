5. Limitations & Caveats

Sample Size Constraints

  • Fleet size: 185 lifts. Statistical power is limited for subgroup analyses (e.g., by brand or type).
  • Door health data: Only ~16 months of losignal data available. Longer time series would improve lag analysis.
  • Positive cases: Faults are relatively rare events. Some analyses may be underpowered.

Data Gaps

  • 7-day rolling tables: trafficlog, loalert, liftseveritylog, eventloggeritem only have last 7 days. Historical traffic patterns could significantly improve predictions.
  • Error code definitions: E2 (dominant error code at 75%) lacks official documentation. Interpretation is inferred from co-occurrence with door faults.
  • Accelerometer data: Not included in this statistical validation (requires per-trip S3 processing). Ride quality grades could add predictive power.

What We Cannot Conclude

  • Causation: Granger causality tests predictive power, not true causation. Error rates may correlate with faults without directly causing them.
  • Generalizability: Results are from a single deployment in Singapore. Different OEMs, climates, or usage patterns may yield different results.
  • Optimal maintenance timing: We can predict which lifts are at risk, but cannot yet prescribe exact maintenance intervals from this data alone.

What Would Strengthen the Analysis

  • Historical traffic data (daily trip counts over years, not just last 7 days)
  • Error code documentation from Chevalier/OEM
  • Maintenance action details (what was replaced, adjusted, etc.)
  • Integration of accelerometer ride quality grades as a predictive feature
  • Larger fleet (500+ lifts) for more robust statistical power
